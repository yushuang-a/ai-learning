RAG（Retrieval-Augmented Generation）技术详解

什么是 RAG？
RAG 是检索增强生成的缩写，它结合了信息检索和文本生成。当用户提出问题后，系统首先从知识库中检索出相关文档片段，然后将这些片段作为上下文提供给 LLM，让 LLM 基于检索到的信息生成回答。

RAG 的核心优势：
1. 解决知识更新问题：不需要重新训练模型，只需更新知识库
2. 解决幻觉问题：LLM 基于检索到的真实文档生成回答
3. 提供可追溯性：可以知道回答基于哪些文档
4. 控制成本：相比微调，RAG 的实现成本更低

RAG 的主要流程：
1. 文档处理：将原始文档分块（Chunking），每块不宜过大或过小
2. Embedding：使用嵌入模型将文本块转为向量
3. 存储：将向量和原始文本存入向量数据库
4. 检索：当用户提问时，将问题转为向量，在向量库中做相似度搜索
5. 生成：将检索到的文本块和原始问题一起交给 LLM 生成回答

常见的 Chunking 策略：
1. 固定长度分块：按字符数或 token 数切分，简单但可能切断语义
2. 递归字符分割：按段落、句子、单词的优先级递归切分
3. 语义分割：根据文本语义变化点做切分，效果最好但最复杂
4. 文档结构分割：利用标题、章节等文档结构做切分

常见的 Embedding 模型：
1. OpenAI text-embedding-3-small/large
2. BGE (BAAI General Embedding)
3. Instructor
4. E5

向量检索方式：
1. 余弦相似度（Cosine Similarity）
2. 欧氏距离
3. 点积相似度

Hybrid Search（混合搜索）：
结合向量检索和关键词检索（如 BM25），在语义相关性和精确匹配之间取得平衡。
