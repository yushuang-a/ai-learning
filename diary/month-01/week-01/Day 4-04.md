# Day 4: Function Calling 工具调用

**日期**：2026-04-24

## 今天做了什么

Function Calling 是今天最大的收获。这是 Agent 的基础——让 LLM 能够调用外部工具。

定义 tools 的方式类似声明式 API：name、description、parameters（JSON Schema）。LLM 返回的不是字符串，而是 `tool_calls` 列表，包含函数名和参数。

实现了一个天气查询和股价查询的 demo。LLM 能根据用户的问题自动选择合适的工具并提取参数。

**对比 Java**：这相当于 LLM 做服务发现 + 参数绑定，然后开发者实现具体逻辑。在 Java 里做同样的事情需要定义接口、参数校验、异常处理、超时重试——这套思维模式在 AI 应用里完全适用，只是调用方从客户端变成了 LLM。
