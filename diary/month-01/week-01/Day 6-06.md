# Day 6: 实战：搭建 CLI 聊天助手

**日期**：2026-04-26

## 今天做了什么

周末把这一周学的内容串起来，写了一个 CLI 聊天助手。

功能很简单：从命令行输入问题，调用 LLM API，支持流式输出，有简单的上下文管理（保存最近 5 轮对话），加上重试和错误处理。

**设计思路**：上下文管理用了一个简单的循环列表，超出长度自动丢弃最早的对话。这个思路和 Java 的 RingBuffer 一样。

**成果**：虽然功能简单，但从零搭一个可用的命令行工具，对理解 OpenAI API 的整体流程很有帮助。接下来要学 Prompt Engineering 了——听说这是 AI 应用开发里最值得深入的技能。
